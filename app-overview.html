Trello Clone

A full-stack project management app — boards, lists, cards, drag-and-drop, labels, checklists, and JWT auth.

React 18 Vite 5 Tailwind CSS Node.js + Express PostgreSQL Prisma ORM JWT Auth Playwright E2E
Section 1

Product

What the app does and the features a user can interact with.

🔐

Authentication

Register with name, email and password. Login issues a short-lived JWT (15 min) kept in memory — never localStorage — plus a 7-day refresh token in an httpOnly cookie for silent re-auth.

🗂️

Boards Dashboard

After login, see a grid of all your boards. Create a board with a title and one of six preset colours. Delete any board (with a confirmation step). New accounts get two pre-filled demo boards automatically.

📋

Lists

Inside a board, add named columns (lists). Rename a list by clicking its title inline. Delete a list — all its cards are removed too. Lists scroll horizontally when there are many.

🃏

Cards

Add cards to any list with a quick inline form. Cards show a colour-coded due-date badge (red = overdue, yellow = today, grey = future) and label colour dots at a glance.

🖱️

Drag & Drop

Drag cards within a list to reorder them, or drop them into a different list. Drag entire lists left and right. Positions are persisted to the server instantly; on failure the UI rolls back.

📝

Card Detail Modal

Click any card to open a full detail panel without leaving the board. Edit the title inline, write a description, set a due date, and close with the × button or by clicking the backdrop.

🏷️

Labels

Inside the card modal, add colour labels with custom text. Labels appear as coloured pills in the modal and as small colour dots on the card tile in the board view. Remove any label with ×.

✅

Checklists

Add one or more named checklists to a card. Each checklist shows a live progress bar (e.g. 3 / 5 items checked). Add, check off, and delete individual items. Delete the whole checklist at once.

React 18 + Vite 5

Single-page app. React Context manages auth state and board state. Axios interceptors automatically attach the Bearer token and retry requests after a silent token refresh.

Node.js + Express 4

REST API on port 4000. All routes under /api/boards, /api/lists, /api/cards are protected by JWT middleware. Input validated with Zod schemas.

Prisma ORM + PostgreSQL

Seven models: User, Board, List, Card, Label, Checklist, ChecklistItem. Cascading deletes ensure referential integrity — deleting a board removes all lists, cards, and their details.

JWT Authentication

Access tokens (15 min) stored in React memory — not localStorage — to prevent XSS. Refresh tokens (7 days) in httpOnly cookies so JavaScript can never read them.

@hello-pangea/dnd

Maintained fork of react-beautiful-dnd. Positions stored as floats — inserting between two items uses (prev + next) / 2, avoiding full re-indexing on every drag.

Tailwind CSS 3

Utility-first styling. Dark header, coloured board backgrounds, white list columns with shadows, modal with backdrop blur, toast notifications for async feedback.

JWT Token Lifecycle

Register / Login

Server returns { accessToken } in the JSON body and sets a refreshToken httpOnly cookie. The React client stores the access token in memory (a module-level variable — never the DOM or localStorage).

Authenticated requests

Every API call goes through the Axios instance, which attaches Authorization: Bearer <accessToken> via a request interceptor.

Token expiry (15 min)

When the server returns 401, the Axios response interceptor automatically calls POST /api/auth/refresh. The browser sends the httpOnly cookie automatically. The server issues a new access token.

Refresh failure

If the refresh token is expired or missing, the interceptor clears the access token from memory and redirects to /login.

Logout

Calls POST /api/auth/logout, which clears the httpOnly cookie server-side. The client clears its in-memory token. User is redirected to /login.

Test database required
