App Overview — Product · Architecture · Tests

Product Features User-facing
🔐

Authentication

  • Register with name, email, password
  • Login with JWT access + refresh tokens
  • Auto-refresh: tokens renew silently on expiry
  • Protected routes redirect unauthenticated users
🗂️

Boards

  • Create boards with title and custom color
  • 6 preset color swatches
  • View all boards in a responsive grid
  • Delete boards (with confirmation)
  • Navigate into a board to manage lists & cards
📄

Lists

  • Add lists to a board
  • Rename lists inline
  • Delete lists
  • Drag & drop to reorder lists within a board
🃏

Cards

  • Add cards to lists
  • Drag & drop between lists and within a list
  • Move cards to different lists
  • Delete cards
  • Open card modal for detailed editing
📝

Card Details (Modal)

  • Inline title editing (click to edit)
  • Rich description textarea
  • Colored labels with custom text
  • Due dates with visual status badge
  • Checklists with progress bar and item toggle
  • Escape key / backdrop click to close
⏰

Due Dates & Status

  • Date picker per card
  • DueDateBadge shows: "Due soon" (yellow) or "Overdue" (red)
  • Badge visible on card tile and inside modal
  • Remove due date with one click

Data Models

  • User — id, email, password (hashed), name
  • Board — id, title, color, ownerId
  • List — id, title, position (Float), boardId
  • Card — id, title, description?, position, dueDate?, listId
  • Label — id, text, color, cardId
  • Checklist — id, title, cardId
  • ChecklistItem — id, text, checked, checklistId

Key Design Decisions

  • Float positions for drag-and-drop — no full re-index on each move; gap inserted between neighbours
  • @hello-pangea/dnd chosen over react-beautiful-dnd (maintained fork, same API)
  • Cascade deletes in Prisma schema — delete a board removes all lists → cards → labels/checklists
  • Zod validates all incoming request bodies server-side
  • Context API only — no Redux/Zustand; auth state is the single global store

Run Tests

cd client && npm test

# Server unit tests
cd server && npm test

# E2E tests (servers must be running)
cd e2e && npx playwright test

Test Configuration

  • Client: client/vite.config.js → vitest config
  • Server: server/jest.config.js → ESM-compatible
  • E2E: e2e/playwright.config.ts
  • Shared test helpers: e2e/fixtures/test-helpers.ts
  • Server setup: server/src/__tests__/setup.js
⚠️
Critical: Separate Test Database Required Server tests truncate ALL tables on each run. Never point NODE_ENV=test at your development database. Configure a separate PostgreSQL DB in server/.env.test.
