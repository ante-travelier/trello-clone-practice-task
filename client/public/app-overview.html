Trello Clone — App Overview

Full-stack Kanban board application with real-time drag-and-drop, JWT auth, and comprehensive test coverage

Features

📋 Boards

  • Create, rename, and delete boards
  • Preset color themes for board backgrounds
  • Dashboard view showing all user boards
  • Board ownership — only the creator can modify

📄 Lists

  • Add unlimited lists to any board
  • Rename and delete lists
  • Drag-and-drop reordering between positions
  • Float-based positioning (no full re-indexing)

🃏 Cards

  • Create cards within lists with title and description
  • Drag cards between lists or reorder within a list
  • Detailed card modal with all metadata
  • Optimistic UI updates with rollback on failure

🏷 Labels

  • Color-coded labels for card categorization
  • Custom label text per card
  • Visual label chips on card tiles
  • Add/remove labels from card modal

☑ Checklists

  • Multiple checklists per card
  • Add, toggle, and remove checklist items
  • Progress tracking per checklist
  • Nested: Card → Checklist → Items

📅 Due Dates

  • Set optional due dates on cards
  • Color-coded badge: green (ok), yellow (soon), red (overdue)
  • Relative time display ("2 hours ago")
  • DateTime picker in card modal

🔒 Authentication

  • Register with name, email, and password
  • JWT access + refresh token flow
  • Access tokens in memory (XSS-safe)
  • Refresh tokens in httpOnly cookies

🔀 Drag & Drop

  • Powered by @hello-pangea/dnd
  • Reorder lists horizontally
  • Move cards between lists
  • Optimistic updates with server sync

System Architecture

Browser (React SPA) │ │ HTTP / REST / JSON │ Auth: Bearer JWT token ▼ Express API Server :4000 │ ├── /api/auth/* Register, Login, Refresh, Logout ├── /api/boards/* CRUD + ownership guard ├── /api/lists/* CRUD + position management ├── /api/cards/* CRUD + details (labels, checklists) │ │ Prisma ORM ▼ PostgreSQL :5432 └── 7 tables: User, Board, List, Card, Label, Checklist, ChecklistItem

Tech Stack

Frontend

  • React 18 — component-based UI
  • Vite 5 — fast dev server & bundler
  • Tailwind CSS 3 — utility-first styling
  • React Router 6 — client-side routing
  • Axios — HTTP client with token interceptors
  • @hello-pangea/dnd — drag-and-drop

Backend

  • Node.js + Express 4 — REST API
  • Prisma ORM — type-safe database access
  • PostgreSQL — relational database
  • bcryptjs — password hashing
  • Zod — input validation schemas
  • jsonwebtoken — JWT token signing

Auth Flow

  • Access token stored in React state (not localStorage)
  • Refresh token in httpOnly cookie (XSS-proof)
  • Axios interceptor auto-refreshes expired tokens
  • Protected routes via JWT middleware
  • Passwords hashed with bcrypt (salt rounds: 10)

Database Schema

ModelKey FieldsRelations
User id, email, password, name, createdAt Has many Board
Board id, title, color, createdAt Belongs to User, has many List
List id, title, position (Float), createdAt Belongs to Board, has many Card
Card id, title, description?, position (Float), dueDate? Belongs to List, has many Label, Checklist
Label id, text, color Belongs to Card
Checklist id, title Belongs to Card, has many ChecklistItem
ChecklistItem id, text, checked (Boolean) Belongs to Checklist

All relations use cascading deletes for referential integrity.

Testing Strategy

🎯 Three-Layer Approach

  • Unit/Component (Client) — isolated React components with mocked APIs, verifying UI rendering and user interactions
  • Integration (Server) — real HTTP requests via Supertest against Express app with test database, verifying business logic and data persistence
  • End-to-End (Playwright) — full browser automation testing complete user journeys across frontend and backend
